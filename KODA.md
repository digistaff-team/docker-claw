# KODA.md — Инструкции для ИИ-агента

## Обзор проекта

**Клиент Завод** (Docker-Claw v3.0.0) — это платформа для запуска изолированных Docker-контейнеров под каждого пользователя. Платформа предоставляет возможность управления окружением через веб-интерфейс, Telegram-бота, Email или HTTP-вебхуки. Встроенный AI-агент способен самостоятельно выполнять команды, анализировать контекст и формировать осмысленные ответы.

### Основные технологии

- **Серверная часть:** Node.js 18+ с Express.js
- **База данных:** PostgreSQL 15 (автоматическое провижирование баз на пользователя)
- **Контейнеризация:** Docker с изоляцией по пользователям
- **Telegram:** Библиотека Telegraf для управления ботами
- **Email:** nodemailer + imap-simple для IMAP/SMTP
- **Фронтенд:** Статические HTML/CSS/JS файлы (без фреймворков)

### Ключевые возможности

- Изолированный sandbox-контейнер для каждого пользователя с ограничением CPU и памяти
- AI-агент в цикле: агент сам решает какие команды запустить, проверяет результаты и отвечает осмысленно
- Мультиканальное управление: Telegram-бот, Email (IMAP/SMTP), HTTP-вебхуки
- Веб-интерфейс для управления сессиями, файлами, задачами и настройками AI
- Персистентное хранилище с авто-бэкапами и многоуровневым undo (снапшоты)
- Персонализация AI через файлы IDENTITY.md, SOUL.md, USER.md, MEMORY.md
- Планировщик контента для автоматической генерации и публикации постов в Telegram-каналы
- Админ-панель для управления всеми контейнерами

## Сборка и запуск

### Установка зависимостей

```bash
npm install
```

### Запуск в режиме разработки

```bash
npm run dev
```

Приложение запустится на порту 3015 с автоматической перезагрузкой при изменениях (используется nodemon).

### Запуск в продакшн

```bash
npm start
```

### Запуск тестов

```bash
npm test
```

Тесты расположены в директории `tests/` и используют встроенный модуль Node.js `assert` (без стороннего тестового фреймворка).

### Docker Compose

Для запуска всех сервисов вместе (PostgreSQL, nginx, приложение):

```bash
docker-compose up -d
```

Просмотр логов:

```bash
docker-compose logs -f app
```

Остановка:

```bash
docker-compose down
```

## Архитектура системы

### Основной поток работы

1. Пользователь проходит аутентификацию через Telegram auth-бот → создаётся сессия → запускается Docker-контейнер (`sandbox-user-{chatId}`)
2. Для каждого пользователя автоматически создаётся персональная PostgreSQL-база данных (`db_{chatId}`)
3. Команды выполняются внутри изолированного контейнера через `/api/execute`
4. AI-агент обрабатывает сообщения через LLM с паттерном tool-calling

### Автоматическое создание базы данных

При создании сессии пользователя (контейнера) автоматически выполняется:

- Создание персональной базы данных PostgreSQL с именем `db_{chatId}`
- Создание необходимых таблиц для системы контента (`content_topics`, `content_materials`, `content_jobs`, `content_posts`, `publish_logs`)
- Это происходит в фоне после инициализации контейнера, в функции `verifyContainerDeps`

Если база данных ещё не существует, она будет создана автоматически при первом обращении к функциям контента.

### Двухботная система Telegram

- **Auth-бот** (`manage/telegram/authBot.js`) — один глобальный экземпляр для авторизации новых пользователей
- **User-боты** (`manage/telegram/runner.js`) — пер-user боты для обработки команд и сообщений

### Система статусов контента

Статусный автомат для контента: `draft → ready → approved → published` (с путями ошибок и неудач).

## Конфигурация

Все переменные окружения централизованы в файле `config.js`. Основные параметры:

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `PORT` | 3015 | Порт HTTP-сервера |
| `PG_HOST` | 172.17.0.1 | Хост PostgreSQL |
| `PG_PASSWORD` | — | Пароль PostgreSQL |
| `DATA_ROOT` | /var/sandbox-data | Корень хранилища пользователей |
| `BACKUP_ROOT` | /var/sandbox-backups | Директория бэкапов |
| `DOCKER_IMAGE` | mcr.microsoft.com/devcontainers/javascript-node:20-bookworm | Образ sandbox-контейнера |
| `CONTAINER_MEMORY` | 256m | Лимит памяти на контейнер (оптимально для сервера с 2GB RAM и 10 пользователями) |
| `CONTAINER_CPUS` | 0.5 | Лимит CPU на контейнер |
| `SESSION_MAX_IDLE_MS` | 86400000 | Таймаут простоя сессии (24 часа) |

Конфигурация загружается из `.env` и `.env.local` (переопределяет).

## Правила разработки

### Стиль кода

- JavaScript (ES6+ модули CommonJS)
- Отступ: 4 пробела
- Именование: camelCase для переменных и функций, PascalCase для классов
- Все комментарии и документация на русском языке

### Тестирование

- Используется встроенный модуль Node.js `assert`
- Тестовые файлы располагаются в директории `tests/`
- Запуск отдельного теста: `node tests/content.status.test.js`

### Безопасность

- Опасные команды (rm -rf /, fork-bomb) заблокированы на уровне сервиса
- Лимит времени выполнения команды — 30 секунд
- Лимит на загрузку файла — 100 МБ
- Доступ к сессии контролируется по chatId

### Frontend

- Статические HTML-страницы в `public/`
- JavaScript в соответствующих файлах в `public/js/`
- Общие утилиты в `public/js/common.js`
- Стили в `public/css/main.css`

## Персонализация AI

Для настройки AI-помощника используются четыре файла внутри контейнера пользователя:

| Файл | Назначение |
|------|------------|
| `IDENTITY.md` | Имя помощника, его роль, чем занимается |
| `SOUL.md` | Как помощник общается: тон, характер, принципы |
| `USER.md` | Информация о пользователе: цели, предпочтения, контекст |
| `MEMORY.md` | Важные данные для работы (доступы, правила) |

Эти файлы хранятся в директории `/workspace` внутри контейнера пользователя.

## Интеграция с ProTalk

Для активации AI-агента необходимы параметры из личного кабинета ProTalk:

- **Bot ID** — идентификатор бота
- **Bot Token** — секретный токен бота
- **Email** — email аккаунта на ProTalk (для проверки баланса)

После настройки Telegram-бот переключается в "умный режим": каждое сообщение обрабатывается LLM, который знает контекст окружения, файлы и персонализацию пользователя.
