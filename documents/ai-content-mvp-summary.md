# Итоговые вводные для MVP: AI-генератор контента и публикация в Telegram

## Статус
Все критичные вводные для реализации MVP собраны и зафиксированы.

## 1. Границы MVP
1. Формат MVP: `text + image`.
2. Видео-генерация: отложена на следующий этап.
3. Режим запуска: по расписанию + ручной запуск.

## 2. Публикация и расписание
1. Таймзона: `Europe/Moscow`.
2. Расписание: ежедневно в `09:00`.
3. Лимит публикаций: 1 пост в сутки.
4. Ручная команда модератора: `/post_now`.

## 3. Telegram-контур
1. Канал публикации: `-1002263032027`.
2. Бот добавлен в канал с правом `Post Messages`.
3. Согласование черновиков: в личный чат модератора.
4. Модератор: один пользователь (`Telegram user id: 128247430`).
5. Модератор уже нажал `/start` у бота.

## 4. Контур согласования
1. Перед публикацией обязателен `Approve`.
2. Состав черновика в модерацию:
   - текст поста;
   - изображение;
   - кнопки: `Approve`, `Regenerate Text`, `Regenerate Image`, `Reject`.
3. Логика `Reject`:
   - автоматическая перегенерация 1 новой версии;
   - если `REJECTED` 3 раза подряд, предложить ручную переработку и ожидать ручного действия.

## 5. Источники контента
1. Темы публикаций: Google Sheets
   - URL: `https://docs.google.com/spreadsheets/d/1klEWbBtbr1Ym-i4Xez76-egXpjBVD8ikiQAfVNyrUjg/edit?usp=sharing`
   - Лист: `План` (`gid=164844003`)
   - Колонки: `Тема`, `Фокусный ключ`, `Вторичные ключи`, `LSI-ключи`, `Статус`
2. Материалы/факты: Google Drive-папка
   - URL: `https://drive.google.com/drive/folders/1Vo7ZmMUR4j2ksZw3LZusPp8oKdmXnO9u?usp=sharing`
3. Политика фактов: использовать только данные из предоставленных файлов/материалов.

## 6. Статусы тем в таблице
1. В работу берется первая строка со `Статус = ""` (пусто).
2. После генерации черновика: `DRAFT_READY`.
3. После `Approve` и публикации: `PUBLISHED`.
4. При ошибке: `FAILED` + текст ошибки.

## 7. Политика контента
1. Язык: русский.
2. Тон: дружелюбно-продающий.
3. Длина: короткий пост до 600 символов.
4. Эмодзи: умеренно.
5. CTA: нужен по контексту публикации.
6. Рубрики:
   - о продукте;
   - о компании;
   - интересные факты.
7. Стоп-темы:
   - война;
   - политика;
   - религия;
   - секс;
   - преступность.
8. Хэштеги:
   - 2-3 ключевых хэштега в конце текста по контексту поста.
9. Фиксированная бренд-подпись: не требуется.

## 8. Генерация изображений
1. Провайдер: OpenAI Images API.
2. Модель: `gpt-image-1`.
3. Ключ: `OPENAI_API_KEY` в `.env`.
4. Лимит попыток генерации изображения: до 3 попыток на пост.

## 9. Бюджет и ограничения
1. Дневной бюджет: `5 USD`.
2. Режим контроля бюджета: `soft warning`.

## 10. Следующий этап реализации
Этап 1 (MVP implementation):
1. БД-схема (`content_jobs`, `content_assets`, `content_posts`, `publish_logs`).
2. Очередь/worker с ретраями и идемпотентностью.
3. Интеграция Google Sheets + Google Drive (read-only, public/shared links).
4. Генерация текста и изображения.
5. Контур Telegram approve.
6. Публикация в Telegram-канал.
7. Обновление статусов в таблице.
