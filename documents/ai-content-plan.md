# План доработки приложения: AI-генератор контента и публикация в Telegram-канал

## Цель
Доработать приложение так, чтобы AI-агент генерировал тексты, изображения и видео, а затем публиковал контент в Telegram-канал через бота в управляемом и безопасном режиме.

## 1. Определить рамки MVP (1-2 дня)
1. Зафиксировать типы контента: `text`, `text+image`, `text+video`.
2. Определить периодичность публикаций, язык, тональность и контент-рубрики.
3. Согласовать бюджет на генерацию и лимиты по количеству публикаций.
4. Выбрать режим: полностью автоматический или через согласование человеком.

## 2. Построить контентный пайплайн (ядро)
1. Добавить сущности в PostgreSQL:
   - `content_jobs`
   - `content_assets`
   - `content_posts`
   - `publish_logs`
2. Ввести статусы жизненного цикла:
   - `draft`
   - `media_generating`
   - `ready`
   - `approved`
   - `published`
   - `failed`
3. Реализовать очередь задач (worker), ретраи и идемпотентность, чтобы исключить дубли публикаций.

## 3. Интегрировать генерацию контента
1. Текст:
   - использовать текущий AI-роутер;
   - добавить отдельные промпт-шаблоны под формат Telegram.
2. Изображения:
   - подключить image API-провайдер;
   - сохранять ассеты в рабочем хранилище.
3. Видео:
   - подключить video API с асинхронным polling;
   - задать повышенные таймауты и ограничение на длительность рендеринга.
4. Fallback-логика:
   - если видео не готово вовремя, публиковать `text+image`.

## 4. Расширить инструменты AI-агента (tools)
1. Добавить инструменты:
   - `generate_post_text`
   - `generate_image`
   - `generate_video`
   - `assemble_post`
   - `publish_to_telegram_channel`
2. Вынести редакционную политику в системные инструкции:
   - длина поста;
   - структура;
   - стиль;
   - ограничения по темам.
3. Включить автопроверки перед публикацией:
   - токсичность/запрещённый контент;
   - валидация длины текста;
   - валидация медиа.

## 5. Реализовать публикацию в Telegram-канал
1. Разделить контуры:
   - бот для управления;
   - бот/механизм публикации в канал.
2. Поддержать форматы отправки:
   - `sendMessage`
   - `sendPhoto`
   - `sendVideo`
   - `sendMediaGroup`
3. Добавить отложенную публикацию по расписанию и принудительную публикацию "сейчас".

## 6. Ввести контур согласования (рекомендуется)
1. Отправлять черновик в приватный чат модератора или веб-панель.
2. Добавить действия:
   - `Approve`
   - `Revise`
   - `Reject`
3. Публиковать в канал только после `Approve`.
4. Сохранять историю версий и причины правок.

## 7. Обеспечить безопасность и наблюдаемость
1. Ввести дневные лимиты по стоимости и количеству генераций.
2. Хранить секреты только в env/secret-store.
3. Добавить метрики и мониторинг:
   - процент успешных публикаций;
   - время генерации;
   - стоимость;
   - ошибки по провайдерам.
4. Настроить алерты на критические сбои и превышение бюджета.

## 8. Рекомендуемый порядок внедрения
1. Этап 1: `text + image + approval` (быстрый и безопасный запуск).
2. Этап 2: подключить видео-генерацию как асинхронный этап.
3. Этап 3: оптимизировать качество контента (A/B, шаблоны рубрик, аналитика).

## Практическая рекомендация
Для старта лучше использовать полуавтоматический режим с обязательным согласованием публикаций. Это снижает репутационные риски, ускоряет выход в прод и позволяет дообучить контентный контур на реальных данных.
