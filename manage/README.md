# Управление окружением через каналы (manage)

Здесь всё, что относится к управлению пользовательским окружением через удобные каналы: Telegram-бот, позже — Email, Slack и т.д.
 
## Telegram-бот  

- **Токен** пользователь вводит в панели (Каналы связи). Сервер сохраняет токен и запускает бота для этого `chat_id`.
- **Верификация**: кто первый напишет боту — получает одноразовый код. Код вводится в панели; после этого только этот Telegram-аккаунт может управлять окружением.
- **Управление**: подтверждённый пользователь пишет боту команды — они выполняются в контейнере пользователя (`sessionService.executeCommand`). Команды `статус` / `status` / `контекст` / `context` возвращают последние команды и структуру файлов.

Файлы:
- `store.js` — состояние (токен, верификация, последние команды), персист в `DATA_ROOT/manage-state.json`.
- `telegram/runner.js` — запуск/остановка ботов (Telegraf), обработка сообщений.
- `telegram/context.js` — сбор контекста (последние команды, файлы, персона).
- `routes.js` — API: сохранение токена/ИИ, верификация, статус, отключение.

## ИИ Ассистент (AI Router)

- **Токен + модель** сохраняются в панели (ИИ → AI Router).
- После настройки Telegram-бот переключается в "умный режим": команды обрабатываются LLM (знает контекст: файлы, команды, персонализацию), использует tool для executeCommand.
- LLM сам решает, какие команды запускать, проверяет результаты, даёт осмысленный ответ.
- **Отправка файлов**: ИИ может отправлять файлы из контейнера (/workspace/output и др.). В ответе LLM указывает `[SEND_FILE: /path/to/file]`, бот парсит маркеры, копирует файл из контейнера и отправляет как document в Telegram.
