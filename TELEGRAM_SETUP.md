# Настройка Telegram бота @DigiStaff_Team_bot

## Проблема
Ошибка "❌ Ошибка соединения с сервером. Попробуйте позже." возникает, когда бот не может подключиться к API серверу.

## Решение

### 1. Проверьте файл `.env`

Файл `.env` должен существовать в корне проекта. Пример содержимого:

```env
# Server
PORT=3015

# API URL - КРИТИЧЕСКИ ВАЖНО!
# Для локальной разработки:
API_URL=http://localhost:3015

# Для продакшена (если сервер на удалённом хосте):
# API_URL=https://claw.pro-talk.ru

# Auth Bot Token (@DigiStaff_Team_bot)
AUTH_BOT_TOKEN=your_bot_token_here

# PostgreSQL
PG_HOST=172.17.0.1
PG_PORT=5432
PG_USER=postgres
PG_PASSWORD=your_password_here
```

### 2. Получите токен бота

1. Откройте Telegram и найдите @BotFather
2. Отправьте команду `/newbot` или выберите существующего бота
3. Скопируйте токен бота
4. Вставьте токен в файл `.env` в переменную `AUTH_BOT_TOKEN`

### 3. Запустите сервер

```powershell
# В PowerShell
cd C:\Projects\Docker-Claw
node server.js
```

Или используйте скрипт:
```powershell
.\run.ps1
```

### 4. Проверьте логи сервера

После запуска сервера проверьте консоль на наличие сообщений:
- `🤖 Auth bot: ✅ STARTED (@DigiStaff_Team_bot)` - бот запущен
- `📡 Server: http://localhost:3015` - сервер слушает порт

### 5. Протестируйте авторизацию

1. Откройте Telegram и найдите @DigiStaff_Team_bot
2. Отправьте команду `/start`
3. Нажмите кнопку "🔑 Войти в аккаунт"
4. Проверьте логи сервера на наличие ошибок

## Диагностика ошибок

### Ошибка: "Сервер недоступен. Убедитесь, что сервер запущен."

**Причина:** Сервер не запущен или не слушает порт 3015.

**Решение:**
1. Запустите сервер: `node server.js`
2. Проверьте, что порт 3015 не занят другой программой

### Ошибка: "Неверный адрес сервера. Проверьте API_URL."

**Причина:** Неправильно указан `API_URL` в файле `.env`.

**Решение:**
1. Для локальной разработки: `API_URL=http://localhost:3015`
2. Для продакшена: `API_URL=https://your-domain.com`

### Ошибка: "Ошибка соединения с сервером" (общая)

**Причина:** Сетевая проблема, firewall, или прокси.

**Решение:**
1. Проверьте, что брандмауэр не блокирует порт 3015
2. Попробуйте добавить в `.env`: `API_URL=http://127.0.0.1:3015`
3. Проверьте логи сервера для подробной информации

### Бот не отвечает на команды

**Причина:** Бот не запущен или токен неверный.

**Решение:**
1. Проверьте, что `AUTH_BOT_TOKEN` указан в `.env`
2. Проверьте, что токен правильный (можно проверить в @BotFather)
3. Перезапустите сервер

## Важно для локальной разработки

Telegram не может напрямую подключиться к `localhost` на вашем компьютере. 
Если вы разрабатываете локально, используйте один из вариантов:

### Вариант 1: Локальный сервер (рекомендуется для тестирования)
```env
API_URL=http://localhost:3015
```
Бот будет вызывать API локально через Node.js fetch.

### Вариант 2: Публичный URL (для продакшена)
```env
API_URL=https://claw.pro-talk.ru
```

### Вариант 3: ngrok (для тестирования с Telegram)
1. Установите ngrok: `npm install -g ngrok`
2. Запустите: `ngrok http 3015`
3. Скопируйте публичный URL (например, `https://abc123.ngrok.io`)
4. Укажите в `.env`: `API_URL=https://abc123.ngrok.io`

## Логи для отладки

После включения подробного логирования в консоль будут выводиться:
- `[AUTH-BOT] Login attempt from ...` - попытка входа
- `[AUTH-BOT] Login error: ...` - ошибка авторизации
- `[AUTH-BOT] API URL: ...` - используемый API URL
- `[TG-LOGIN-ERROR] ...` - ошибки входа в пользовательском боте

## Контакты

Если проблема не решена, проверьте логи сервера и обратитесь в поддержку.
